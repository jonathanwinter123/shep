# Updating the App Icon

## Overview

The icon pipeline: SVG source → 1024px PNG → all platform icons → Tauri embeds at compile time.

## Files

| File | Purpose |
|---|---|
| `assets/shep_logo.svg` | Source logo artwork |
| `scripts/generate-icon.mjs` | Renders SVG onto a styled 1024px background |
| `assets/icon-1024.png` | Generated master icon (input to `tauri icon`) |
| `src-tauri/icons/` | All platform-specific icons (generated by Tauri CLI) |

## Steps

### 1. Edit the SVG

Modify `assets/shep_logo.svg` with the new design. The generate script reads
the SVG's `viewBox` to scale it automatically.

To adjust padding, corner radius, or background colors, edit `scripts/generate-icon.mjs`.

### 2. Generate the master PNG

```bash
node scripts/generate-icon.mjs
```

This writes `assets/icon-1024.png`. Open it to verify the result looks correct.

### 3. Generate platform icons

```bash
pnpm tauri icon assets/icon-1024.png
```

This overwrites everything in `src-tauri/icons/` (macOS `.icns`, Windows `.ico`,
PNGs at all required sizes, iOS and Android icons).

### 4. Clean the Rust build and recompile

Tauri embeds icons at compile time via `tauri::generate_context!()`. Cargo does
not track icon files as dependencies, so changing them won't trigger a rebuild
on its own. You must clean first:

```bash
cargo clean --manifest-path src-tauri/Cargo.toml
pnpm tauri dev
```

Without `cargo clean`, the dev build will keep showing the old icon.

### 5. Clear macOS icon caches (if needed)

macOS aggressively caches app icons. If you still see the old icon after a
clean rebuild, clear the caches:

```bash
# Find your cache folder (the hash segment varies per machine)
find /private/var/folders -name "com.apple.iconservices" -o -name "com.apple.dock.iconcache" -o -name "com.apple.iconservicesagent" 2>/dev/null

# Remove them
rm -rf /private/var/folders/<your-path>/C/com.apple.iconservicesagent
rm -rf /private/var/folders/<your-path>/C/com.apple.iconservices
rm -rf /private/var/folders/<your-path>/C/com.apple.dock.iconcache

# Restart Dock and Finder
killall Dock
killall Finder
```

If a previous release build exists at `src-tauri/target/release/bundle/`, macOS
Launch Services may have registered its icon. Remove the stale bundle:

```bash
rm -rf src-tauri/target/release/bundle
```

## Quick Reference

Full pipeline from scratch:

```bash
node scripts/generate-icon.mjs
pnpm tauri icon assets/icon-1024.png
cargo clean --manifest-path src-tauri/Cargo.toml
pnpm tauri dev
```
